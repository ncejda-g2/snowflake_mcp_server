⏺ Final Implementation Plan for Snowflake MCP Server

  Architecture Overview

  Snowflake MCP Server
  ├── Catalog Refresh Tool (pre-indexes all schemas with 5-day cache)
  ├── Schema Inspector Tool (reads from cache)
  ├── Table Schema Tool (gets detailed column info)
  ├── Query Executor Tool (runs read-only queries with double safety)
  └── Automatic Result Formatter (built into query executor)

  Files to Create/Modify:

  1. pyproject.toml

  - Rename to snowflake_mcp_server
  - Add snowflake-connector-python, cachetools
  - Keep fastmcp, httpx, pydantic
  - Update metadata

  2. server/config.py

  class Config(BaseModel):
      # Snowflake connection
      account: str = Field(description="Snowflake account identifier")
      username: str = Field(description="Snowflake username")
      warehouse: str = Field(description="Compute warehouse")
      token: str = Field(description="Snowflake PAT", exclude=True)

      # MCP server settings
      host: str = "0.0.0.0"
      port: int = 8000
      transport: str = "stdio"  # or "http"

      # Cache settings
      cache_ttl_days: int = 5
      max_query_rows: int = 100

      # Safety settings
      # ALWAYS ALWAYS ALWAYS enforce read_only. NEVER allow this as a boolean option.
      debug: bool = False

  3. server/snowflake_connection.py (NEW)

  class SnowflakeConnection:
      - Connect using PAT authentication
      - Enforce read-only at session level
      - Set AUTOCOMMIT=FALSE, BEGIN TRANSACTION READ ONLY
      - Validate no write permissions
      - Query logging

  4. server/schema_cache.py (NEW)

  class SchemaCache:
      - TTL-based cache (5 days)
      - Store database->schema->table metadata
      - Automatic invalidation/refresh
      - Persist cache to disk for restarts

  5. server/tools/ (REPLACE ALL)

  Delete all G2 tools, create:

  - catalog_refresh.py
  @mcp.tool
  async def refresh_catalog():
      """Refresh the schema catalog cache by scanning all accessible databases"""
      - Query INFORMATION_SCHEMA across all databases
      - Build comprehensive metadata index
      - Store in cache with timestamp
      - Return summary of discovered schemas
  - schema_inspector.py
  @mcp.tool
  async def inspect_schemas(database_pattern: str = None):
      """List available databases, schemas, and tables"""
      - Check cache validity (auto-refresh if expired)
      - Filter by pattern if provided
      - Return hierarchical structure
  - table_inspector.py
  @mcp.tool
  async def get_table_schema(database: str, schema: str, table: str):
      """Get detailed column information for a specific table"""
      - Fetch column names, types, constraints
      - Include sample data if requested
      - Return formatted schema info
  - query_executor.py
  @mcp.tool
  async def execute_query(sql: str, database: str = None):
      """Execute read-only SQL query with safety checks"""
      - Parse SQL to reject write operations
      - Execute in read-only transaction
      - Auto-format results
      - Implement pagination (100 rows default)
      - Log all queries
      - Return formatted results
      - Require the refresh_catalog results to be in cache before running queries.

  6. server/sql_validator.py 

  class SQLValidator:
      - Parse SQL using sqlparse or similar
      - Reject INSERT/UPDATE/DELETE/CREATE/DROP/ALTER/MERGE/TRUNCATE
      - Validate only SELECT/WITH/SHOW/DESCRIBE allowed
      - Return validation errors with helpful messages

  7. server/app.py

  from fastmcp import FastMCP
  from server.snowflake_connection import SnowflakeConnection
  from server.tools import (
      catalog_refresh, schema_inspector,
      table_inspector, query_executor
  )

  mcp = FastMCP("Snowflake Read-Only MCP")

  # Initialize connection on startup
  conn = SnowflakeConnection(config)

  # Register tools
  mcp.tool(catalog_refresh.refresh_catalog)
  mcp.tool(schema_inspector.inspect_schemas)
  mcp.tool(table_inspector.get_table_schema)
  mcp.tool(query_executor.execute_query)

  Safety Measures

  1. Double enforcement: SQL parsing + Snowflake session settings
  2. No write operations: Reject at multiple levels
  3. Query logging: Full audit trail
  4. Connection validation: Verify read-only on connect
  5. Error messages: Clear feedback when writes attempted
